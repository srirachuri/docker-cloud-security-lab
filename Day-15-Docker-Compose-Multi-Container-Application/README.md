# 🐳 Day 15 – Docker Compose Multi-Container Application

## 📌 Objective

Learn how to use Docker Compose to define, deploy, manage, and remove multiple containers with a single configuration file. This project demonstrates running an Nginx web server and a MySQL database together using Docker Compose.

## 📄 compose.yaml

```
services:
  web:
    image: nginx
    container_name: myweb
    ports:
      - "8080:80"

  db:
    image: mysql:8.0
    container_name: mydb
    environment:
      MYSQL_ROOT_PASSWORD: password123
```

---

## 📝 Configuration Explanation

### Web Service

- Uses the official **Nginx** image.
- Creates a container named **myweb**.
- Maps port **8080** on the host to port **80** inside the container.

---

### Database Service

- Uses the official **MySQL 8.0** image.
- Creates a container named **mydb**.
- Sets the MySQL root password using an environment variable.

# ⚡ Commands Used

## Create Project Directory

```
mkdir docker_compose
cd docker_compose
```

---

## Create Compose File

```
notepad compose.yaml
```

---

## Validate Configuration

```
docker compose config
```

Checks whether the Compose file is valid.

---

## Start Containers

```
docker compose up -d
```

Creates the network, starts the Nginx and MySQL containers, and runs them in detached mode.

---

## View Running Containers

```
docker ps
```

Lists all running containers.

---

## View Nginx Logs

```
docker compose logs web
```

Displays logs generated by the web service.

---

## View MySQL Logs

```
docker compose logs db
```

Displays logs generated by the database service.

---

## Check Compose Services

```
docker compose ps
```

Shows the status of services managed by Docker Compose.

---

## List Docker Volumes

```
docker volume ls
```

Displays all Docker volumes created by Compose.

---

## List Docker Networks

```
docker network ls
```

Displays available Docker networks.

---

## Stop Services

```
docker compose stop
```

Stops all running services without removing them.

---

## Start Services Again

```
docker compose start
```

Starts previously stopped services.

---

## Remove Containers, Network, and Volumes

```
docker compose down -v
```

Stops and removes containers, networks, and associated volumes.

---

## Verify Removal

```
docker compose ps -a
```

Confirms that all Compose-managed containers have been removed.
