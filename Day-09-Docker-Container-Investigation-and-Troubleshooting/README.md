## ⚡ Commands Used

### View Running Containers

```
docker ps
```

Displays all currently running containers with:

- Container ID
- Image
- Status
- Port Mapping
- Container Name

---

### Check Port Mapping

```
docker port <container-id>
```

Example:

```
docker port 4edf3cf5e9ae
```

Output:

```
80/tcp -> 0.0.0.0:8081
80/tcp -> [::]:8081
```

This verifies that host port **8081** is mapped to container port **80**.

---

### View Container Logs

```
docker logs <container-id>
```

Example:

```
docker logs 4edf3cf5e9ae
```

This displays:

- Container startup process
- Nginx initialisation
- Worker process creation
- Configuration loading

---

### Follow Logs in Real Time

```
docker logs -f <container-id>
```

Example:

```
docker logs -f 4edf3cf5e9ae
```

Used to monitor live application activity including:

- Browser requests
- HTTP status codes
- Error messages
- Access logs

---

### Inspect Container Configuration

```
docker inspect <container-id>
```

Example:

```
docker inspect 4edf3cf5e9ae
```

Provides detailed information such as:

- Container ID
- Running status
- Network configuration
- Port bindings
- Mounted volumes
- Log configuration
- Restart policy
- Environment details

---

### Stop a Running Container

```
docker stop <container-id>
```

Example:

```
docker stop 4edf3cf5e9ae
```

Gracefully stops the running container.

## 🔒 Key Learning

Docker provides powerful investigation commands that help administrators troubleshoot running containers.

The `docker inspect` command offers detailed configuration information, while `docker logs` and `docker logs -f` help monitor application behaviour and identify issues. Together, these commands are essential for diagnosing container problems and understanding runtime behaviour.
